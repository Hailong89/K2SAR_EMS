Supported Operating Systems

This tool work under Unix (and compatible) and Win32 systems, and requires the Perl interpreter v5.8.0 or greater. It should also be portable without problem under others OS , but that was not tested. If you are interested, ask the Authors.

Command line options

The “WarnungsFilter” run in console mode (Note: do you really think that it runs in graphic mode under Windows®?)
It accepts two types of arguments: the simple arguments (like --help) and the arguments with a value (like --verbosity=3)
Without argument, the script will display the following message:
WarnungsFilter [Version.Release] - type WarnungsFilter.pl --help for help .

If the same argument is specified several times, the last one overwrites the others.

The followings arguments are recognized (but in the actual version, some are not implemented), and all are case sensitive

  • --help
    Displays the help and exit.
    The displayed message can be found under [Script's directory]/Doc/help.txt.
    If specified, all the others arguments are ignored (Because if you need help, you probably don't want to process the warnings immediately)

  • --version
    Displays the version of the tool, and of each module used by the tool, and exit
    If specified, all the others arguments are ignored (except --help, because it has the highest priority)

  • --history
    Displays the history of the tool, and exit
    If specified, all the others arguments are ignored (except --help and --version, because they have the highest priority)

  • --silent
    Do not display anything on the output, except error message (when the script dies, or crashes for example)
    it means, nothing is displayed on STDOUT
    this does not apply for the tool's log file. (not available. Will be done later)

  • --verbosity=n (default = 2)
    Verbosity level. It applies for the output's log file (and STDOUT, except if –silent is turned on)
    0 -> Does not say anything (in this case, no output's log file generated)
    1 -> Only says what it's supposed to do (generating this, or that...)
    2 -> Gives some statistics when processing, and all the name of the files generated
    3 -> Gives number of lines processed, and others details
    4 -> Gives details about intern routines, results of intermediate computes
    5 -> Says all. You probably don't want this (except if you are the developer himself)

  • --sort=n (default = no sorting)
    Sort the categories into different ways:
    1 -> Sorted by number of files.
    2 -> Sorted by warnings.
    3 -> Sorted by occurences.
    4 -> Sorted by names.
    other value: -> Sorted by the internal hash function of the perl interpreter.

  • --input-file=File (no default: must be specified somewhere, here or in a Configuration file)
    Specifies the input file to process, with the extension.
    If the directory is omitted, the current work directory is used

  • --log-file=File (default = [Tool's Out Directory]/Wf.log) (not implemented yet)
    Gives the name of the tool's log file. If you don't want a log file, look at this option:    --verbosity

  • --config-file=File (default = no configuration file)
    Specifies the configuration file to use. If the path is omitted or relative, the script will try to found the specified configuration file in the following directories, in this order:
    - [current directory]/
    -[home directory]/
    - [home directory]/.WarnungsFilter/
    - [Script's directory]/templates/Config/
    if the file is not found, the script dies with the message “The configuration file [name of the file] was not found in the following directories [list of directories]”

  • --OS=Operating_System (default = auto detected from the $^O built-in variable)
    Please choose in the list (case sensitive!)
    - VMS
    - MSDOS (You shouldn't use this, because of the long names, i.e the extension “html” has to many characters... Don't report bug for this.)
    - MacOS
    - AmigaOS
    - MSWin32 (applies for all the MS Windows® verions, since the Windows® 95)
    - Unix
    - linux (is identical to Unix)
    If you are giving something else, Unix is assumed (Sorry, but the module used to choice the OS belongs to standard libraries of Perl. If you don't like this, specify the option correctly, don't disturb the author)
    Only MSWin32 and Unix (or linux) where tested. If you are interested in other operating systems, ask the authors. The script will probably work with the others system, but it's not sure.
    This option might be removed soon, because at the moment the path is problematic. This will be fixed in new versions.

  • --out-dir=Directory (default = [Work Directory]/.auto_wf/
    If this is a relative path, it will be take regarding the current work directory
    Do not forget the last “/” (or “\” for MS style path)

  • --warning-url-database (default = [Tool's directory]/Help/Url_Database.pl)
    This mini perl-script contains two hashes, named %WF_Warning_Help_Link and %WF_Warning_Help_Text. It gives the tool the url to the html help files for the warning's categories. For more details, look at the explanations in the template Perl script.

Explanations:

I suppose you won't type each time all the details in the line command. (But you are free to do it) That's why this tool can be configured by a configuration file. The syntax of a configuration file is very simple... Just be careful with the names: they are case-sensitive (because the developer uses Linux, and because the Perl language is case-sensitive. Don't you see the difference between “a” and “A”? If you don't, you are MS formatted. If you see a difference, you are a Human)

A section is a single word, quoted by “[“ and “]” without spaces. All the key that follows the section belongs to it. If there is more than one section with the same name, the keys will be concatenated. The tool will only read the keys that are in the section [WarnungsFilter]
Example:
[WarnungsFilter]
[Hello_World]
This won't work:
[ Hello]
[Hello World]

A key is a single word, followed by a “=” . All that follows the = (except first spaces) is the value of the key. If the script find more identical keys (same section, same name) the last one found overwrite the others.
Examples:
Bill=Gates
Bill= Gates
Bill = Gates
The key “Bill” will get the value “Gates”
test = sdefds {gd)dfh{'%
the key test will get the value “sdefds {gd)dfh{'%”. If you want to play with this tool, you can try it. But you are not paid to play!
test a = 23
This returns an error.
If a key doesn't belong to a section, it returns an error.

You can add a comment, like this: A comment is a line, where the first non-spacing character is a “#”.
Examples:
#This is a comment
# This is a comment too
Bill = Gates # this is not a comment
In the last example, the key “Bill” will get the value “ Gates # this is not a comment”. It' not the expected result. Why is it so? There are two reasons: the first is because you may want use the “#” string in a key, and the second is that the developer don't want to write a complicated script to detect the first “#” in the chain.

An empty line, or a line containing only spaces or tabs is ignored. Did you expect something else?

A configuration file is a text file. It means that it shoud always be ended by an empty line. (like the C and H sources, hmm?) This shouldn't be a problem for the script (I hope).

KeyWords

First, you have to begin with the tool's specific section:
[WarnungsFilter]

The name of the keys are the same than the command line options, without the “--” at the beginning. The eventual “-” in the name of the option must be replaced by a “_” (because “-” does not belong to a word, but “_” does). If a key is not reconized by the tool, it is ignored. Some command line option can not be used in the configuration file, like “help”, “version”, “history”, and “config_file”: they will be ignored (there is probably a good reason for this... Do you know which one?)

The keys are the following:

  • silent
    value: 0 or 1 (if something else, assume 0)

  • verbosity

  • input_file
    (must be specified somewhere!!!)

  • OS

  • out_dir

  • log_file

Examples

Only one example is provided. Because you will be soon able to do it yourself.
The script is susceptible: if the syntax of the configuration file is not correct, it stops.

# Syntax of the configuration file:
# The # at beginning of line mean "this is a comment"
# empty lines are ignored (of course...)
# the form of the configuration file is:
# [section]
# key = value
# All the names are case sensitive!!!

# if there is two sections with the same name,
# they are concatenated
# if there is two or more identical keys in the same section,
# the last one overwrite the others.

[All]
author = The free Electron

[WarnungsFilter]
input_log_file = Warning.log
# Do not forget the last "/"
out_dir = makeout\WF\
verbosity = 2
log_file = wf.log
# A very bad choice:
OS = MSWin32
