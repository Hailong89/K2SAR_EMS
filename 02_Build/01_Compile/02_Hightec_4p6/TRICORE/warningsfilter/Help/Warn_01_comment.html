"/*" within comment
Not dangerous

Problem

A begin of commentary /* is following a non terminated previous /*, or you tried to use nested commentaries (in the case you probably get an error before.)


Solution

Always use a matching pair of /* and */ in your code. The comments are working by pair, but they can't be nested.


Example

The compiler warns on the red "begin of commentary", but the really problem concerns the non terminated previous one (blue)

/************************************************************/
/* calculate unlimited value of dwmsvo_w                    */
/* use Int16 addition against overflow
/* dwmsvvst_w -3276.7...3276.8 16 Bit 0.1 Grad KW           */
/* DWMSVRDR -3276.7...3276.8 16 Bit 0.1 Grad KW             */
/* (legal range: -360/SY_CAMNMSV ... +360/SY_CAMNMSV)       */
/************************************************************/

The correction is very simple(blue):

/************************************************************/
/* calculate unlimited value of dwmsvo_w                    */
/* use Int16 addition against overflow                      */
/* dwmsvvst_w -3276.7...3276.8 16 Bit 0.1 Grad KW           */
/* DWMSVRDR -3276.7...3276.8 16 Bit 0.1 Grad KW             */
/* (legal range: -360/SY_CAMNMSV ... +360/SY_CAMNMSV)       */
/************************************************************/
